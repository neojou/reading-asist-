# AI 學習小組：AI 輔助深度閱讀與思考系統

![AI Reading Team](./images/AI_reading_team.png)  

**AI 學習小組** 是一個模擬多角色專家團隊的 AI 系統，專門幫助人類對**書籍、論文、文章、長文**進行系統級深度閱讀與批判思考。透過多位 AI 角色分工合作，將異質資料、跨領域知識與機制性證據整合，最終產出「可行動的個人知識資產」。

這個專案的核心是 **AGENTS.md**，它定義了完整的角色、責任、流程與操作規則。您只需將 AGENTS.md 內容貼到 ChatGPT、Claude、Grok 等大型語言模型的對話中，就能立即啟動這個「AI 閱讀小隊」。

[AI 讀書小組 - 和 AI 組隊一起讀書 - 2026.2.19](https://njiot.blogspot.com/2026/02/ai-ai-2026219.html)

## 核心特色

- **多角色協作**：Coordinator、Explainer、Retriever、Integrator、Affirmer、Critic、Synthesizer、Writer、DevOps 等角色，從不同視角深度剖析內容
- **輕自動 + 手動控制**：Owner 輸入文章與目標後，只由 Coordinator 先提供初步介紹與操作指南，其餘角色需手動切換，精準控制討論深度
- **可行動輸出**：最終產生結構化報告，包括時間軸、知識圖譜、正反論點、新假說、實務應用提案
- **跨平台通用**：支援 ChatGPT、Claude、Grok 等模型，無需額外程式碼
- **持續優化**：內建 DevOps 角色，可討論並改進小隊流程與效率

## 快速開始

1. 開啟一個新的對話（ChatGPT、Claude 或 Grok）
2. 把整個 **AGENTS.md** 內容完整貼上去  
   （或輸入：「請載入 AGENTS.md」後貼上）
3. 以 Owner 身份啟動閱讀：

   ```
   Owner - https://arxiv.org/pdf/xxxx.pdf
   閱讀目標：深度理解這篇論文的核心機制，並思考如何應用到我的研究/投資/產品設計中
   ```

   或

   ```
   Owner - [直接貼上文章全文或重點段落]
   閱讀目標：xxx + 模式：手動（可選，完全手動控制）
   ```

4. Coordinator 會自動回應：
   - 確認文章是否可讀（連結會嘗試瀏覽）
   - 根據目標建議討論順序
   - 提供手動切換角色指南

5. 開始手動切換角色，例如：

   ```
   Explainer: 請給大眾版 + 專業版摘要，並畫結構圖
   Retriever: 找出 20 篇最相關的後續/反面研究
   Critic: 針對核心主張給出最嚴苛的批評
   Synthesizer: 整合所有討論，提出 3–5 個可測試新假說
   Writer: 匯總目前所有關鍵洞見成一份簡潔報告
   ```

6. 討論結束時：

   ```
   Coordinator: 請輸出最終結構化報告
   ```

   或直接說「結束閱讀」

## 團隊角色一覽

| 組別             | 角色          | 主要責任簡述                              |
|------------------|---------------|--------------------------------------------|
| Owner           | Owner        | 提供內容、目標、決策者（人類）            |
| 閱讀解說組      | Explainer    | 三層解說 + 逐段拆解                       |
|                 | Verifier     | 逐句事實查核                              |
| 資料蒐集與整合組| Retriever    | 搜尋相關文獻                              |
|                 | Integrator   | 建時間軸與知識圖譜                        |
| 辯證分析組      | Affirmer     | 最強正面證據                              |
|                 | Critic       | 最嚴苛反面批評                            |
| 昇華創新組      | Synthesizer  | 跨域整合 + 新假說                         |
|                 | Coordinator  | 計劃制定、協調、最終報告                 |
|                 | Writer       | 討論匯總與精煉摘要                        |
| 系統優化組      | DevOps       | 分析並優化小隊流程與效率                  |

詳細角色責任、輸出規範、注意事項請見 **[AGENTS.md](./AGENTS.md)**

## 使用建議

- **適合場景**：論文閱讀、書籍精讀、投資報告分析、領域新知快速上手、批判性思考訓練
- **提示技巧**：閱讀目標越具體，輸出品質越好（e.g. 「應用到我的軟體架構決策」比「理解這篇論文」更好）
- **進階玩法**：用 `Writer:` 定期匯總；用 `DevOps:` 討論如何改進流程；跨對話用 conversation search 整合歷史討論
